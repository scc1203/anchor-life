# ⚓️ Anchor 全息宇宙系统架构白皮书 (The Architecture Bible)
**当前适用版本：v7.1.3 及以上 (Cloud & IoT Era)**

## 1. 核心产品哲学 (Core Philosophy)
- **产品定位**：这不是一个 KPI 监工，而是专为 INTJ/INFJ 及高敏感人群设计的“生活重启装置”与“数字修道院”。
- **三时岸理念**：以旧岸为归处（历史足迹），以此刻为渡船（抽卡与专注），以晨昕为应许（未来灵感）。
- **极简与克制 (Calm Tech)**：
  - 用“随机抽卡”对抗决策瘫痪，用“物理锚点”（打钩/正计时）代替大脑记忆。
  - 允许烂尾，允许休息。绝对不加机械死板的 DDL（截止日期）报警，绝不在主卡片堆砌多余按钮。
  - 视觉保持马卡龙色系、毛玻璃质感、大面积留白。

## 2. 空间架构：一厅四室与外设 (Spatial & Phygital UI)
目前的系统由纯前端 HTML/JS 驱动，通过 NAS API 实现多端与实体同步。
- **01区 控舵大厅**：主舞台。极度纯净，仅保留能量药丸（Uptime/Anchor/Downtime）与抽卡机。
- **02区 进货口**：无摩擦的灵感捕手与兵工厂（单次任务与专属清单的创建地），带有极强的洗稿与自动打标引擎。
- **03区 岁月局**：档案馆。包含任务库、媒体库、冷库、足迹、历史，以及顶部的“🟩 岁月热力图”和“📁 专属档案柜”。
- **04区 底舱机房**：系统偏好、夜猫子时钟、起飞着陆仪式、云端机房（NAS 同步）与数据方舟。
- **00区 穹顶静室**：情绪糖果区，解决极度瘫痪时的感官启动。
- **IoT 实体结界**：支持 M5Stack 等物理小方块读取云端状态，实现桌面物理抽卡与计时联动。

## 3. 🚨 核心暗逻辑与防御装甲 (Crucial Dark Logic - DO NOT BREAK)
在进行任何 UI 重构、代码搬迁或新功能开发时，**必须誓死捍卫**以下底层逻辑：

### 3.1 防刷新丢失装甲 (The Live State Armor)
- 依赖 `localStorage.getItem(KEY_LIVE_STATE)`。任务一旦被抽出并进入 `anchor` (进行中) 状态，必须将 `currentTask`、`startTime` 以及所属的 `activePlaylist`（如果有）写入本地缓存。
- **铁律**：页面刷新时，必须完美还原卡片 UI 和**正计时秒表**，绝不允许任务丢失或状态机断层。

### 3.2 专属清单生命周期 (Playlist Lifecycle)
- **⚡️ Once (一次性/今日待办)**：卡片端或档案柜打满 100% 后，必须向足迹写入 `🏆【整体完成】`，并**自动从档案柜彻底销毁**（阅后即焚）。
- **🔄 SOP (标准作业程序)**：打满 100% 后，必须写入 `🏆【+X】` 足迹，将完成次数 `completeCount + 1`，并重置所有子任务勾选态，永久保留。
- **🚀 长期项目 (Project)**：进度满格后触发自动杀青机制，杜绝复活死循环。
- **隧道心流**：清单抽卡必须支持“连抽”（drawFromPlaylist），在卡片上点击“完成子任务”必须与底层的档案柜实现**双向状态同步**。

### 3.3 记忆时光机与无损轮回 (Data Reincarnation & Time Machine)
- **无损复活**：历史足迹或荣誉殿堂中的任务，支持“一键提取至任务库”。复活时必须抹除原有的死亡印记（完成日期），带回所有的时长和书签快照，实现真正的新生。
- **时光机修正**：在历史记录中修改时长或进度时，必须保证数据安全无损回调。
- **手动归档**：书/影/剧等媒体任务，日常打卡只记足迹；只有在菜单中点击「🏆 归档」时，才允许正式进入荣誉殿堂，严禁日常打卡刷屏殿堂。

### 3.4 隐形管家与时空法则 (Smart Engine & Time Rules)
- **夜猫子时钟**：系统中所有的 `new Date()` 判天逻辑，必须通过 `getAnchorDate()` 处理。跨天结算严格遵循用户在 04 区设置的“一日开启时间”。历史记录需绘制“🌙 跨日线”。
- **洗稿机引擎 (autoTag)**：02区输入时，严格遵循首字或特定动词（逛、赏、演出等）的智能拦截，优先判定 `culture` 媒体库，并在入库前拍摄 `finalIcon` 快照，绝不允许后续渲染时图标被篡改。
- **1分钟微启动**：允许用户破冰尝试。1分钟结束后点完成，记为 `🐾` 或 `🎬` 标记；若转正，**必须继承**这1分钟的倒计时动能，不得从零开始。
- **一朵花（灵魂锚点）**：具有独立姿态的里程碑事件，在热力图和足迹中拥有最高视觉优先级。

## 4. 🚨 给 AI 开发者的最高行动宪法 (Data Security Baseline)
1. **数据神圣不可侵犯 (Backward Compatibility)**：严禁擅自删除或重命名 `db`, `archive`, `dailyLog`, `customPlaylists`, `KEY_USERNAME` 等任何 localStorage 的 Key。新增功能必须保证旧版 JSON 导入时能静默补齐默认值，绝对不允许引发白屏或报错！
2. **极简代码 (No Bloatware)**：禁止随意引入庞大的第三方框架或动画库，坚持使用原生 HTML/CSS/JS 实现功能。
3. **注释要求 (Explain Why & What)**：所有新增的 JavaScript 逻辑，必须包含详尽的 `// 注释`，用通俗的语言向主理人解释这段代码在干什么。
4. **切忌一刀切 (Context Awareness)**：在修改任何导致状态机变化的代码（如任务流转、状态重置、关闭 Overlay）时，必须全局考量已有设定。遇到深水区，务必在修改前自行排查 `completeTaskCore` 和 `changeTask` 的关联逻辑，给出最安全的缝合方案！
