            04 · 用户手册 & 数据方舟
            📖 用户手册 & 逻辑词典
            🚀 3 分钟上手（按区走一遍）
            1. 02 · 新任务入库： 把打算做的事输入进去（可批量输入）。 可选 🏷️标签、📍地点、⏳时长； 🔁 每日循环。
            2. 00 · 吃一颗糖： 先做 30 秒“轻动作”（🫧气泡 / 🖍️涂鸦 / 🎵音乐等），从停滞切到可行动。
            3. 01 · 抽签装置： 点击 🔮 帮我选抽取下一个任务。 抽到后只做一个决定：✅ 完成 / 🔁 换一个。
            4. （可选）📝 记忆： 抽到结果卡右上角点 📝 备注，记录书签/时长/吐槽（不强制填）。
            5. 03 · 任务库 & 记录： 需要手动处理时，在 任务库/媒体库 点击任务名称，会弹出 完成 / 修改 / 记忆。
            6. 04 · 数据方舟： 定期 📥 导出 备份；换设备或误删时用 📤 导入 恢复。
            📚 逻辑词典（点我展开）
            🎴 01 · 抽签装置（筛选 + 抽卡）
            • ☀️ Uptime（推进）： 更偏“推进、清单、关键任务”。通常不会推剧集/唱片/SOS；如果你有 🐸 青蛙，会明显更优先。
            • ⚓ Anchor（均衡）： 最“公平”的混合抽取，适合大多数时刻。
            • ☁️ Downtime（休息）： 更偏“轻松、输入、恢复”。通常不会推 🐸 青蛙；更容易出现剧集/电影/唱片等。
            • 🔽 高级筛选： 只在你想控制变量时再开（避免把抽卡变成填写表格）。
            📝 记忆模块（书签 / 本次投入 / 自由备注）

            入口只有两个：① 抽卡结果卡右上角「📝 备注」；② 任务库/媒体库点任务名 →「📝 记忆」。

            一、📍书签进度（坐标型，覆盖最新）
            • 适合：书页数、剧集集数、电影时间点。
            • 写法示例：P120 / S2E5 / 00:42:10
            • 规则：每次保存会覆盖上一次书签；下次抽到该任务，会提示“上次进度”。
            二、⏳本次投入时长（累积型，加总）
            • 适合：练习/学习/项目推进等“投入越多越有成就”的事。
            • 用法：在时长选择器里选 0h 45m 这类值，然后保存。
            • 规则：每次保存都会累积到总时长；结果卡会显示“累计总时长”（若你开启了显示）。
            三、🗒️自由备注（主观型，可写链接/评价/提醒）
            • 适合：吐槽、复盘一句话、材料链接、下次怎么继续。
            • 显示策略：列表区不展开（避免拥挤）；抽到/点记忆时再看。

            设计原则：不强迫你每次填写。记忆只在“你需要上下文时”出现。

            🗂️ 03 · 任务库 / 媒体库（如何操作一条任务）
            • 点击任务名称：弹出操作菜单（✅ 完成 / ✏️ 修改 / 📝 记忆）。
            • ✅ 完成：进入 👣 今日足迹 与 📅 历史记录。
            • ✏️ 修改：改标题、标签、时长、循环等（用于纠错与维护任务库）。
            • 📝 记忆：同抽卡页的记忆面板，用来补进度/时长/备注。
            ⌨️ 智能输入词典（少点点点，多说人话）

            支持批量粘贴：每行一个任务。系统会根据关键词自动补标签/建议时长等。

            • 🎬📖 书影剧自动归类： 用 看/读/书/影/剧/追 开头会走文化逻辑（自动书名号等）。
            • ⚡️ 速办任务： 输入洗/刷/理/拆/扔/喝等 ，会建议 ⚡️ 速办（5m 级别）。
            • 🐸 青蛙任务： 用 */!/重要 + 空格 开头，标记为更难但更重要的一类（更容易在 ☀️Uptime 出现）。
            • 🔁 每日循环： 输入含 每天/每日，自动开启循环（适合习惯/保底任务）。

            建议：先“粗糙入库”，再靠抽卡推进；不要在入库阶段把自己卡死在完美分类。

            🧊 冷库（暂时退场，不再被抽到）
            • 适合：当前阶段不想碰、但又不舍得删的任务。
            • 进入冷库后：不会被抽到；你可以在冷库里 ♻️ 赦免回库 或 🗑️ 彻底删除。
            📅 今日足迹 & 历史记录（记录会显示什么）
            • 👣 今日足迹：只保留当天完成的清单，方便“今天我做过什么”。
            • 🏆 荣誉殿堂： 更偏“永久里程碑”：看完/读完/追完的文化内容，以及重要任务的完成记录会沉淀在这里。
            • 📅 历史记录：按日期汇总完成记录，适合回顾与复盘。
            • 📝 记忆显示： 如果你在完成/抽卡时记录了「⏳本次时长 / 📍书签进度 / 🗒️备注」， 历史记录会把它们跟在该条任务后面（过长会只显示第一行/截断）。
            🌅 起飞 / 🌙 着陆 / 🆘 SOS（低门槛重启）
            • 🌅 起床起飞：开始一天/开始一段工作时，用一套固定动作把状态拉起来。
            • 🌙 睡前着陆：睡前或结束时，用一套固定动作把大脑慢慢关机。
            • 怎么编辑： 页面底部有 📝 起飞 / 着陆仪式编辑，每行一个步骤；留空会自动恢复默认模板。
            • 🆘 SOS：只给“最小动作”（呼吸/喝水/伸展等），目的是止损与重启，不是逼你高强度推进。
            📊 计数器：🛡️ 意志力 / ✨ 小确幸
            • 🛡️ 意志力： 想做坏习惯但忍住时点一下，给“克制”留证据。
            • ✨ 小确幸： 遇到开心小事点一下，给“好事”留记号。
            • 用法建议： 不要当 KPI；它更像“你在活着”的低摩擦记录。
            📡 数据方舟（备份 / 导入 / 容量说明）
            • 建议频率：每周或每次大版本更新前，📥 导出 一次备份（JSON）。
            • 换设备 / 误删：用 📤 导入 恢复。
            • 容量提示：本项目数据存在浏览器 localStorage，空间有限；历史记录越长、备注越多，占用越快。
